Instruções para um Projeto GIT (GITHUB)

Clone: O comando "git clone (URL)", como dito previamente em sua prória sintaxe serve para "clonar", ou seja, recriar um repositório remoto já existente, seja vazio ou não.

Branch: O comando "git branch (NOME)" é utilizado para criar ramificações, ramificações essas que servem para separar um projeto em diversas áreas e juntá-las no final.

Commit: O comando "git commit" que geralmente é continuado pelos parâmetros '-m "mensagem"' serve para armazenar as mudanças feitas no projeto, no entanto, os parâmetros servem para seguir o armazenamento de uma mensagem.

Merge: O merge é utilizado para finalmente mesclar as Branches criadas anteriormente no comando "git branch"

Pull Request: Os pull requests são pedidos que o desenvolvedor efetua através de seu terminal/hud para adquirir a atualização mais recente do projeto, evitando conflitos desnecessários. Na maioria dos casos, o pull request tem que ser autorizado pelo gestor.
